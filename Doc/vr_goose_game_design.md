# 🪿 VR 鵝頭偷麵包遊戲 — 遊戲設計文件

> **對應開發規劃：** [`vr_goose_dev_plan.md`](vr_goose_dev_plan.md) v2.1  
> **文件版本：** v2.1　**最後更新：** 2026-05-03

---

## 核心概念

**背景故事：** 小鵝喜歡吃麵包；玩家扮演的鵝要去**偷麵包**。進入中後段關卡後，場上會出現**游來游去的小鵝**，玩家除了搬麵包，還要把牠們**抓回巢裡**。

**操作：** 玩家用手模擬鵝頭，控制嘴部開合與頭部轉向，伸長脖子把**麵包**或**小鵝**帶回巢中（與開發文件中的 Snap Grabbing、Nest 雙類型計數一致）。

---

## 🗺️ 關卡設計

> 統一欄位：**場景配置 / 關卡提示詞（英文 UI） / 訓練重點 / 玩法說明 / 過關條件 / 失敗條件 / 核心機制 / ⚠️ 待補強**  
> **註：** 麵包外觀（吐司、長棍等）可由美術共用多種 Prefab，不再綁定「第幾關必須是哪一種」。  
> **關卡提示詞：** 給 UI／字幕／教學條用的短英文句；實作時可由 `LevelConfig` 或本地化表帶入。

### 第零關：起始畫面 — 擺麵包開局

| 項目 | 內容 |
|------|------|
| **場景配置** | 巢在視線可及處；巢內或旁邊有**小鵝**（待機／氛圍）；旁邊有 **1 塊可抓取麵包**（或清楚可辨的「開始道具」） |
| **關卡提示詞** | Put **1** bread in the nest to start. |
| **訓練重點** | 理解「巢」是目標區；與正式關卡相同的**放進巢裡**操作 |
| **玩法說明** | 把麵包咬起或抓起，送入巢的判定範圍，觸發**開始遊戲**（進入第一關） |
| **過關條件** | 1 塊麵包成功計入巢中（僅麵包，不強制把小鵝也算進目標） |
| **失敗條件** | 可選：麵包落水 → Fail UI → 重置；或第零關刻意不做失敗以減少摩擦 |
| **核心機制** | 與 Nest 共用；`breadToWin = 1`、`gooseToWin = 0`；開場 UI 引導 |
| **⚠️ 待補強** | 「把麵包放進巢裡開始」圖示或短文案；第零關巢內小鵝**不應**被誤計入關卡鵝隻（程式需排除或分 Tag／Layer） |

### 第一關：鉤子 + 高處麵包 + 平台雙麵包

| 項目 | 內容 |
|------|------|
| **場景配置** | **1 塊麵包**在**較高處**（需用**長鉤子**勾近再咬）；**平台（或兩處平台）上共 2 塊麵包** |
| **關卡提示詞** | Put **3** breads in the nest to pass the level. |
| **訓練重點** | 伸長、咬取、縮回；**工具（鉤子）**延伸觸及範圍；近距與遠距目標切換 |
| **玩法說明** | 高處麵包：拿鉤子 → 鉤頭碰觸帶回 → 近嘴後改嘴咬；平台上兩塊可直接咬或用鉤子輔助（依場景擺放） |
| **過關條件** | 巢中累計 **3 塊麵包**；順序不限；無時間限制 |
| **失敗條件** | 任一麵包掉入水中 → 暫停 + Fail UI → 重來（建議三塊一併重置） |
| **核心機制** | 長鉤子實體物理帶動麵包 + Snap 咬取；巢僅計麵包 `breadDelivered == 3` |
| **⚠️ 待補強** | 鉤頭 Collider、麵包質量／阻力 playtesting；新手箭頭是否僅本關顯示 |

---

### 第二關：移動平台上的雙麵包 + 兩隻小鵝

| 項目 | 內容 |
|------|------|
| **場景配置** | **2 塊麵包**置於**移動平台**（1～2 座平台承載，場上共 2 麵包為準）；**2 隻小鵝**在**水面**或其它活動區游動 |
| **關卡提示詞** | Put **2** breads and take **2** little geese to the nest to pass the level. |
| **訓練重點** | **預判移動麵包**時機；同一關內**麵包 + 活體小鵝**兩類目標 |
| **玩法說明** | 在平台經過有利位置時咬住麵包回巢；小鵝用法與後續關卡相同（嘴／手抓取帶回巢） |
| **過關條件** | **2 麵包 + 2 小鵝**皆曾進入巢並各計一次（順序不限） |
| **失敗條件** | 麵包落水 → Fail + 重置；小鵝落水建議重生或不判 Fail |
| **核心機制** | `MovingPlatform` + Nest `breadDelivered` / `gooseDelivered`；小鵝 AI：游動、扶正、LookAt |
| **⚠️ 待補強** | 平台速度與咬取容錯；兩鵝同場難度曲線 |

### 第三關：中央巢 + 雙高腳桌 + 刷子推麵包

| 項目 | 內容 |
|------|------|
| **場景配置** | **巢**在場景**中央**，周邊圍起動線；**2 張高腳桌**各 **1 塊麵包**；**刷子**（長柄推物）將麵包**推入巢** Trigger |
| **關卡提示詞** | Put **2** breads in the nest to pass the level. *(Tip: Use the brush to push bread.)* |
| **訓練重點** | 非直接用嘴舉高的路線——**工具推物**、桌面高度與落點 |
| **玩法說明** | 抓取刷子推擠麵包滑落／滑向巢；亦可視實作允許鉤子／嘴輔助（主軸仍是刷子） |
| **過關條件** | **2 塊麵包**皆進巢並計入 |
| **失敗條件** | 麵包落水 → Fail + 重置（與專案一致） |
| **核心機制** | 刷子 `Grabbable` + 物理碰撞推麵包；`breadToWin = 2`、`gooseToWin = 0` |
| **⚠️ 待補強** | 刷子 Prefab、桌面摩擦與防穿透；巢周碰撞避免卡麵包 |

### 第四關：五隻小鵝

| 項目 | 內容 |
|------|------|
| **場景配置** | **5 隻小鵝**於水面（或其它安全活動區）游動 |
| **關卡提示詞** | Take **5** little geese to the nest to pass the level. |
| **訓練重點** | 多目標連續回收；維持追視與舒適度 |
| **玩法說明** | 逐隻把小鵝送回巢；每隻只計一次 |
| **過關條件** | **5 隻小鵝**皆回巢 |
| **失敗條件** | 無麵包則不適用麵包落水；小鵝落水規則同第二關建議 |
| **核心機制** | `gooseDelivered >= 5`、`breadToWin = 0` |
| **⚠️ 待補強** | 五隻同場效能／穿模；游速與範圍調節 |

### 第五關：三高腳桌三麵包 + 一隻小鵝

| 項目 | 內容 |
|------|------|
| **場景配置** | **3 張高腳桌**各 **1 塊麵包**；**1 隻小鵝**活動（水面或其它） |
| **關卡提示詞** | Put **3** breads and take **1** little goose to the nest to pass the level. |
| **訓練重點** | 高腳桌連續目標 + 單隻小鵝分心項 |
| **玩法說明** | 麵包可咬取、鉤取或（若場景保留）刷子輔助；小鵝抓回巢 |
| **過關條件** | **3 麵包 + 1 小鵝**皆回巢 |
| **失敗條件** | 麵包落水 → Fail + 重置 |
| **核心機制** | Nest 雙計數；企劃可選第三關同款刷子或簡化為鉤子 |
| **⚠️ 待補強** | 桌面高度與玩家頸伸舒適度 |

### 第六關：四座移動平台四塊麵包

| 項目 | 內容 |
|------|------|
| **場景配置** | **4 座移動平台**，各 **1 塊麵包**（軌跡可錯相以增加難度） |
| **關卡提示詞** | Put **4** breads in the nest to pass the level. |
| **訓練重點** | **多平台預判**、連續咬取與返巢 |
| **玩法說明** | 等各平台時機咬住麵包；順序自訂 |
| **過關條件** | **4 塊麵包**皆回巢 |
| **失敗條件** | 任一麵包落水 → Fail + 重置 |
| **核心機制** | 多實例 `MovingPlatform`；`breadToWin = 4` |
| **⚠️ 待補強** | 平台數多時的暈眩與難度梯隊 |

### 第七關：來回移動巢 + 三隻小鵝

| 項目 | 內容 |
|------|------|
| **場景配置** | **巢**（含 Trigger）沿路徑**來回移動**；**3 隻小鵝**場內活動 |
| **關卡提示詞** | Take **3** little geese to the nest to pass the level. *(The nest moves!)* |
| **訓練重點** | **對準移動中的巢**交付；與小鵝 AI 疊加的走位 |
| **玩法說明** | 叼著小鵝追上巢的 Trigger 範圍送達；巢位移時判定需隨 Parent 一致 |
| **過關條件** | **3 隻小鵝**皆送達並計入 |
| **失敗條件** | 小鵝落水規則同前；麵包本關通常為 0 |
| **核心機制** | `nestMoves`／移動巢 + `gooseDelivered >= 3` |
| **⚠️ 待補強** | 巢速與 VR 舒適度（與槍手勢位移、頸伸疊加評估） |

### 🔮 後續更新（非主線）：巡邏與視線躲避

舊版曾規劃**巡邏員 + 圓錐光束**：手（鵝頭）進入光束判定為「被發現」。此玩法**不列入當前主線**，保留給之後版本擴充；細節見開發文件中的 `Guard`、Layer 與 `hasPatrolGuard` 預設關閉說明。

## 🐤 小鵝體驗要點（與程式對齊）

| 面向 | 設計意圖 |
|------|---------|
| **動畫** | 待機、游水循環；可選：被叼起、入巢短演出 |
| **游動** | 水面範圍內巡航，避免呆站；難度由速度與轉向頻率調節 |
| **落地扶正** | 若被甩到岸上翻倒，自動回正，避免「死魚躺」破壞可愛感 |
| **頭部反應** | 玩家鵝頭靠近到一定距離內，小鵝**臉／頭**朝向玩家（LookAt，可限角度），增加「被發現、在看你」的趣味 |
| **與巢** | 進巢只計一次；與麵包分開計數，避免與第零關「巢內裝飾鵝」混淆（程式層需排除） |

## 🛠️ VR 技術實作建議（與開發文件一致）

### 1. Snap Grabbing

- **問題：** VR 難以精準「碰到」小物件。  
- **解決方案：** Meta Interaction SDK 的 Hand Grab + 吸附到嘴部錨點；**小鵝**可共用同一套抓取鏈，另加 AI 與動畫。  
- **效果：** 降低挫折感，動作直覺。

### 2. 巢的判定 — The Nest

- **問題：** 收手時容易與身體或攝影機干涉。  
- **解決方案：** 巢使用較大的 Trigger 球體；麵包／小鵝進入後解除嘴部 Parent、可選短動畫移入堆疊位置。  
- **v2 擴充：** 巢需分辨 **麵包** 與 **小鵝**，分別累計，**兩條件都達成**才過關（依關卡 `breadToWin` / `gooseToWin`）。第七關巢位移時 Trigger 須隨巢移動。

### 3. 物理堆疊

- 抓取時 `isKinematic = true`；放開或入巢後再依需求開啟物理。  
- 巢底需 Collider 承接堆疊。

### 4. 實體長鉤子 — Long Hook（第一關）

- 可抓取的整根鉤子，鉤頭以**實體碰撞**帶動麵包；縮手拉回後改嘴咬。  
- **第一關**為高處 + 平台麵包的核心工具。

### 5. 移動平台（第二、六關等）

- 建議平滑往返（如正弦軌跡）；速度需 playtesting，避免過快導致暈眩或過難。第六關可多實例並行。

### 6. 刷子與高腳桌（第三、五關）

- 刷子為獨立可抓取物，以碰撞對麵布施力；桌面高度與巢周圍需避免卡死。

### 7. 隨機縮放與旋轉 — Visual Variety

- 麵包生成時可隨機 Scale（例如 0.9～1.1）與輕微旋轉，堆在巢中較自然。  
- 小鵝可在外觀或配件上做少量變化（選配）。

## 麵包與美術配置（參考）

不再強制「第 N 關 = 固定麵包種類」。美術可提供多種麵包 Prefab，由關卡設計師在場景中擺放；若需行銷截圖，可仍以吐司、長棍、甜甜圈等混搭。
