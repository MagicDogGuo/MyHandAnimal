# 🪿 VR 鵝頭偷麵包遊戲 — 遊戲設計文件

> **對應開發規劃：** [`vr_goose_dev_plan.md`](vr_goose_dev_plan.md) v2.0  
> **文件版本：** v2.0　**最後更新：** 2026-04-24

---

## 核心概念

**背景故事：** 小鵝喜歡吃麵包；玩家扮演的鵝要去**偷麵包**。進入中後段關卡後，場上會出現**游來游去的小鵝**，玩家除了搬麵包，還要把牠們**抓回巢裡**。

**操作：** 玩家用手模擬鵝頭，控制嘴部開合與頭部轉向，伸長脖子把**麵包**或**小鵝**帶回巢中（與開發文件中的 Snap Grabbing、Nest 雙類型計數一致）。

---

## 🗺️ 關卡設計

> 統一欄位：**場景配置 / 訓練重點 / 玩法說明 / 過關條件 / 失敗條件 / 核心機制 / ⚠️ 待補強**  
> **註：** 麵包外觀（吐司、長棍等）可由美術共用多種 Prefab，不再綁定「第幾關必須是哪一種」。

### 第零關：起始畫面 — 擺麵包開局

| 項目 | 內容 |
|------|------|
| **場景配置** | 巢在視線可及處；巢內或旁邊有**小鵝**（待機／氛圍）；旁邊有 **1 塊可抓取麵包**（或清楚可辨的「開始道具」） |
| **訓練重點** | 理解「巢」是目標區；與正式關卡相同的**放進巢裡**操作 |
| **玩法說明** | 把麵包咬起或抓起，送入巢的判定範圍，觸發**開始遊戲**（進入第一關） |
| **過關條件** | 1 塊麵包成功計入巢中（僅麵包，不強制把小鵝也算進目標） |
| **失敗條件** | 可選：麵包落水 → Fail UI → 重置；或第零關刻意不做失敗以減少摩擦 |
| **核心機制** | 與 Nest 共用；`breadToWin = 1`、`gooseToWin = 0`；開場 UI 引導 |
| **⚠️ 待補強** | 「把麵包放進巢裡開始」圖示或短文案；第零關巢內小鵝**不應**被誤計入關卡鵝隻（程式需排除或分 Tag／Layer） |

---

### 第一關：鉤子 + 高處麵包 + 平台雙麵包

| 項目 | 內容 |
|------|------|
| **場景配置** | **1 塊麵包**在**較高處**（需用**長鉤子**勾近再咬）；**平台（或兩處平台）上共 2 塊麵包** |
| **訓練重點** | 伸長、咬取、縮回；**工具（鉤子）**延伸觸及範圍；近距與遠距目標切換 |
| **玩法說明** | 高處麵包：拿鉤子 → 鉤頭碰觸帶回 → 近嘴後改嘴咬；平台上兩塊可直接咬或用鉤子輔助（依場景擺放） |
| **過關條件** | 巢中累計 **3 塊麵包**；順序不限；無時間限制 |
| **失敗條件** | 任一麵包掉入水中 → 暫停 + Fail UI → 重來（建議三塊一併重置） |
| **核心機制** | 長鉤子實體物理帶動麵包 + Snap 咬取；巢僅計麵包 `breadDelivered == 3` |
| **⚠️ 待補強** | 鉤頭 Collider、麵包質量／阻力 playtesting；新手箭頭是否僅本關顯示 |

---

### 第二關：兩塊麵包 + 一隻小鵝

| 項目 | 內容 |
|------|------|
| **場景配置** | **2 塊麵包**；**1 隻小鵝**在**水面**四處游動 |
| **訓練重點** | 同一關內**兩類目標**（麵包 + 活體小鵝）；追擊移動中的鵝 |
| **玩法說明** | 麵包與小鵝皆可用嘴／手抓取規則帶回巢；小鵝會游動，需預判位置 |
| **過關條件** | **2 麵包 + 1 小鵝**皆曾進入巢並各計一次（順序不限） |
| **失敗條件** | 麵包落水 → Fail + 重置；小鵝是否因落水失敗由企劃定（建議：小鵝以水面活動為主，落水重生或不判 Fail） |
| **核心機制** | Nest 分別累加 `breadDelivered` / `gooseDelivered`；小鵝 AI：游動、落地扶正、靠近玩家時頭部 LookAt（見開發文件） |
| **⚠️ 待補強** | 小鵝被抓時動畫與物理切換；難度曲線（游速、活動範圍） |

---

### 第三關：三隻小鵝回巢

| 項目 | 內容 |
|------|------|
| **場景配置** | **3 隻小鵝**在水面游動（可分散生成） |
| **訓練重點** | 連續追蹤多隻移動目標；重複「抓回巢」循環 |
| **玩法說明** | 逐隻把小鵝送回巢；每隻只計一次 |
| **過關條件** | **3 隻小鵝**皆回巢 |
| **失敗條件** | 依專案一致原則（麵包本關若無則不適用）；小鵝落水規則同第二關建議 |
| **核心機制** | `gooseDelivered >= 3`；AI 同第二關 |
| **⚠️ 待補強** | 三隻同場時效能與導航是否穿模；玩家眩暈感（追視移動物） |

---

### 第四關：移動平台麵包 + 三隻小鵝

| 項目 | 內容 |
|------|------|
| **場景配置** | **移動平台**上放 **1 塊麵包**（左右或往返移動）；**3 隻小鵝**水面游動 |
| **訓練重點** | **預判移動麵包**的時機 + 第三關的多鵝回收 |
| **玩法說明** | 在平台有利位置咬住麵包並回巢；其餘時間抓小鵝；順序自訂 |
| **過關條件** | **1 麵包 + 3 小鵝**皆回巢 |
| **失敗條件** | 麵包落水 → Fail + 重置；其餘同前 |
| **核心機制** | `MovingPlatform`（如 Sine 往返）+ Nest 雙計數 |
| **⚠️ 待補強** | 平台速度與軌跡 playtesting；移動中咬取的容錯（Snap 範圍） |

## 🐤 小鵝體驗要點（與程式對齊）

| 面向 | 設計意圖 |
|------|---------|
| **動畫** | 待機、游水循環；可選：被叼起、入巢短演出 |
| **游動** | 水面範圍內巡航，避免呆站；難度由速度與轉向頻率調節 |
| **落地扶正** | 若被甩到岸上翻倒，自動回正，避免「死魚躺」破壞可愛感 |
| **頭部反應** | 玩家鵝頭靠近到一定距離內，小鵝**臉／頭**朝向玩家（LookAt，可限角度），增加「被發現、在看你」的趣味 |
| **與巢** | 進巢只計一次；與麵包分開計數，避免與第零關「巢內裝飾鵝」混淆（程式層需排除） |

## 🛠️ VR 技術實作建議（與開發文件一致）

### 1. Snap Grabbing

- **問題：** VR 難以精準「碰到」小物件。  
- **解決方案：** Meta Interaction SDK 的 Hand Grab + 吸附到嘴部錨點；**小鵝**可共用同一套抓取鏈，另加 AI 與動畫。  
- **效果：** 降低挫折感，動作直覺。

### 2. 巢的判定 — The Nest

- **問題：** 收手時容易與身體或攝影機干涉。  
- **解決方案：** 巢使用較大的 Trigger 球體；麵包／小鵝進入後解除嘴部 Parent、可選短動畫移入堆疊位置。  
- **v2 擴充：** 巢需分辨 **麵包** 與 **小鵝**，分別累計，**兩條件都達成**才過關（依關卡 `breadToWin` / `gooseToWin`）。

### 3. 物理堆疊

- 抓取時 `isKinematic = true`；放開或入巢後再依需求開啟物理。  
- 巢底需 Collider 承接堆疊。

### 4. 實體長鉤子 — Long Hook（第一關）

- 可抓取的整根鉤子，鉤頭以**實體碰撞**帶動麵包；縮手拉回後改嘴咬。  
- **第一關**為高處 + 平台麵包的核心工具。

### 5. 移動平台（第四關）

- 建議平滑往返（如正弦軌跡）；速度需 playtesting，避免過快導致暈眩或過難。

### 6. 隨機縮放與旋轉 — Visual Variety

- 麵包生成時可隨機 Scale（例如 0.9～1.1）與輕微旋轉，堆在巢中較自然。  
- 小鵝可在外觀或配件上做少量變化（選配）。

## 麵包與美術配置（參考）

不再強制「第 N 關 = 固定麵包種類」。美術可提供多種麵包 Prefab，由關卡設計師在場景中擺放；若需行銷截圖，可仍以吐司、長棍、甜甜圈等混搭。
