# 🪿 VR 鵝頭偷麵包遊戲 — 遊戲設計文件

> **核心概念：** 玩家用手模擬鵝頭，控制嘴部開合與頭部轉向，伸出脖子偷麵包帶回巢中。

---

## 🗺️ 關卡設計

> 統一欄位：**場景配置 / 訓練重點 / 玩法說明 / 過關條件 / 失敗條件 / 核心機制 / 麵包種類 / ⚠️ 待補強**

---

### 第一關：新手送餐 *Straight to the Nest*

| 項目 | 內容 |
|------|------|
| **場景配置** | 麵包就在正前方的碼頭邊緣，巢在玩家視線正下方 |
| **訓練重點** | 練習「伸出 → 咬住 → 縮回」的完整路徑 |
| **玩法說明** | 直線伸出鵝頭咬住麵包，縮回放入巢中 |
| **過關條件** | 成功將麵包放入巢中 1 次 |
| **失敗條件** | 麵包掉入水中 → 暫停 + Fail UI → 重來 |
| **核心機制** | 基礎 Snap Grabbing；無障礙、無時限 |
| **麵包種類** | 🍞 白吐司 |
| **⚠️ 待補強** | 缺少新手引導 UI（例如箭頭提示伸手方向） |

---

### 第二關：多重囤糧 *Multitasking*

| 項目 | 內容 |
|------|------|
| **場景配置** | 岸邊散落 3 塊麵包（左、中、右），間距需足夠讓玩家明顯感受到橫移 |
| **訓練重點** | 練習橫向掃描（X 軸移動） |
| **玩法說明** | 來回三次，依序把三塊麵包全部抓回巢裡 |
| **過關條件** | `foodCount == 3` 時顯示「Level Clear!」；抓取順序不限；無時間限制 |
| **失敗條件** | 任一麵包掉入水中 → 暫停 + Fail UI → 重來（三塊全部重置） |
| **核心機制** | 巢內整數計數器 `int foodCount`；麵包抓取順序不限 |
| **麵包種類** | 🥖 長棍麵包 |
| **⚠️ 待補強** | 無 |

---

### 第三關：移動的目標 *The Moving Meal*

| 項目 | 內容 |
|------|------|
| **場景配置** | 麵包放在一個左右緩慢來回滑動的平台上（掃街機器人 / 推車） |
| **訓練重點** | 練習預判移動目標、掌握抓取時機 |
| **玩法說明** | 類似「打地鼠」——在麵包移動到有利位置的瞬間伸出鵝頭咬住 |
| **過關條件** | 成功將麵包放入巢中 1 次；無時間限制 |
| **失敗條件** | 麵包掉入水中 → 暫停 + Fail UI → 重來 |
| **核心機制** | 平台以固定速度或 Sine 曲線往返；脖子為純視覺，不參與物理碰撞；咬空無懲罰 |
| **麵包種類** | 🍩 甜甜圈 |
| **⚠️ 待補強** | 平台移動速度數值未定義（需 playtesting 校準） |

---

### 第四關：巡邏員 *視線躲避*

| 項目 | 內容 |
|------|------|
| **場景配置** | 岸邊有麵包，巡邏員持手電筒或頭部發出圓錐光束來回巡視 |
| **訓練重點** | 空間感知，保護鵝頭（手）不進入光束範圍 |
| **玩法說明** | 趁光束轉開的空隙伸出鵝頭咬麵包，光束掃回前縮手 |
| **過關條件** | 成功將麵包放入巢中 3 次且未被發現；無時間限制 |
| **失敗條件** | 手（鵝頭）進入光束 → 暫停 + Fail UI → 重來；麵包掉入水中亦同 |
| **核心機制** | 紅綠燈系統：光束為 Cone Trigger，手進入即判定「被發現」；脖子穿模不計入判定 |
| **麵包種類** | 🥐 可頌 |
| **⚠️ 待補強** | 巡邏員的巡邏路徑、轉頭速度未定義（需 playtesting 校準）；是否有「警戒狀態」過渡動畫待決定 |

---

### 第五關：移動目標 + 巡邏 *時機抓取*

| 項目 | 內容 |
|------|------|
| **場景配置** | 岸邊有柱子 / 箱子作為遮蔽物；麵包在移動平台上；巡邏員持續掃視 |
| **訓練重點** | 綜合：預判移動目標 + 躲避光束，在空檔內完成抓取 |
| **玩法說明** | 躲在遮蔽物後，趁巡邏員轉頭的空檔，閃電伸出鵝頭咬住移動中的麵包，再縮回遮蔽物 |
| **過關條件** | 成功將麵包放入巢中 3 次且未被發現；無時間限制 |
| **失敗條件** | 手（鵝頭）進入光束 → 暫停 + Fail UI → 重來；麵包掉入水中亦同 |
| **核心機制** | 安全區系統：遮蔽物後方 / 水面下方視為安全；結合第三關（移動平台）＋第四關（光束判定）機制；咬空無懲罰 |
| **麵包種類** | 🍞🥖🍩 混合隨機 |
| **⚠️ 待補強** | 遮蔽物位置與光束掃描路徑需實際 playtesting 確認難度平衡 |

## 🛠️ VR 技術實作建議

### 1. 解決手感問題 — Snap Grabbing

- **問題：** VR 裡用鵝嘴「精準碰到」小麵包困難，缺乏觸覺回饋
- **解決方案：** 給鵝頭（手）一個稍大的 Trigger Zone，當手靠近麵包約 **5–10 公分**時，麵包自動吸附（Snap）到鵝嘴上
- **效果：** 大幅降低玩家挫折感

---

### 2. 巢的物理判定 — The Nest

- **問題：** 玩家收手放麵包時，手可能撞到自己的身體或攝影機
- **解決方案：** 將「巢」的判定區域放大——只要麵包進入玩家胸口下方的隱形大球體，即自動取消麵包的 Parent 讓它掉落，或播放「吸入巢中」的小位移動畫

---

### 3. 物理堆疊法 — Physics Pile

當鵝嘴（手）將麵包帶到巢的上方時，放開按鈕（或鬆開捏合），麵包恢復物理模擬。

```
抓取時：isKinematic = true   // 跟著手走
放開時：isKinematic = false  // 受重力掉落
```

- 巢的底部加 `Mesh Collider` 或 `Box Collider` 接住麵包
- **效果：** 麵包自然堆疊在巢裡，視覺上更真實

---

### 4. 隨機縮放與旋轉 — Visual Variety

- **目的：** Low Poly 風格下，避免麵包看起來像複製品
- **做法：** 麵包 Spawn 時加入隨機 Rotation 與 Scale（例如 **0.9 ~ 1.1 倍**）
- **效果：** 堆在巢裡時看起來更像一堆真實食物

**各關麵包種類建議：**

| 關卡 | 麵包種類 |
|------|---------|
| 第一關 | 白吐司 |
| 第二關 | 長棍麵包 |
| 第三關 | 甜甜圈 |
| 第四關 | 可頌 |
| 第五關 | 混合隨機 |
